# Доступные Паттерны

## 1. `PatternType.SINGLE`

**Описание:** Один блок - только тот блок, на который кликнул игрок.

**Отображение:** "Single"

**Использование:**
```java
new ToolHoe(new Item.Properties().stacksTo(1), PatternType.SINGLE)
```

**Схема:**
```
[X]  ← Кликнутый блок
```

---

## 2. `PatternType.THREE`

**Описание:** 3 блока в линию по направлению взгляда игрока. Блок, на который кликнул игрок, является точкой отсчета.

**Отображение:** "Line 3"

**Использование:**
```java
new ToolHoe(new Item.Properties().stacksTo(1), PatternType.THREE)
```

**Схема:**
```
[X] → [2] → [3]  ← По направлению взгляда
```

---

## 3. `PatternType.FIVE`

**Описание:** 5 блоков в линию по направлению взгляда игрока.

**Отображение:** "Line 5"

**Использование:**
```java
new ToolHoe(new Item.Properties().stacksTo(1), PatternType.FIVE)
```

**Схема:**
```
[X] → [2] → [3] → [4] → [5]  ← По направлению взгляда
```

---

## 4. `PatternType.GRID_3X3`

**Описание:** Квадрат 3x3, центр - блок на который кликнул игрок.

**Отображение:** "Grid 3x3"

**Схема (вид сверху):**
```
[ ][ ][ ]
[ ][X][ ]  (X = origin)
[ ][ ][ ]
```

**Всего блоков:** 9

---

## 5. `PatternType.GRID_5X5`

**Описание:** Квадрат 5x5, центр - блок на который кликнул игрок.

**Отображение:** "Grid 5x5"

**Схема (вид сверху):**
```
[ ][ ][ ][ ][ ]
[ ][ ][ ][ ][ ]
[ ][ ][X][ ][ ]  (X = origin)
[ ][ ][ ][ ][ ]
[ ][ ][ ][ ][ ]
```

**Всего блоков:** 25

# Переключение Паттернов

## 🎮 Быстрый Старт

### Как переключать паттерны инструментов:
1. Возьмите инструмент в руку (мотыгу или лейку)
2. Зажмите **Shift** + нажмите **ПКМ (правая кнопка мыши)**
3. Паттерн переключится на следующий доступный
4. Вы увидите сообщение с названием нового паттерна

> ⚠️ **Важно**: При зажатом Shift инструмент **не будет** копать землю или поливать грядки - только переключать паттерн!

### Где посмотреть текущий паттерн:
- Наведите курсор на инструмент в инвентаре
- В описании (tooltip) будет показан текущий паттерн
- Например: `Pattern: Single` или `Pattern: Line 3`

## 💡 Полезные Советы

1. **Настройка сохраняется**: Выбранный паттерн сохраняется в предмете, даже если вы положите его в сундук
2. **Звуковая обратная связь**: При переключении слышен клик, подтверждающий смену паттерна
3. **Сообщение в action bar**: Название нового паттерна показывается над хотбаром
4. **Экономия ресурсов**: Используйте паттерн `Single` когда нужно обработать только один блок
5. **Эффективность**: Переключайтесь на больший паттерн для быстрой обработки больших участков
6. **Безопасное переключение**: С зажатым Shift можно безопасно кликать по блокам

# Визуализация Паттернов

## 🎮 Как Это Работает

### В Игре:
1. Возьмите инструмент с паттерном в руку (мотыгу или лейку)
2. Наведите прицел на **грядку (farmland) или землю (dirt)**
3. Увидите **цветной оверлей** на всех блоках, которые затронет паттерн:
   - 🟢 **Зелёный** = блок можно обработать (вспахать/полить)
   - 🔴 **Красный** = блок нельзя обработать

### ⚠️ Важно:
- Визуализация **появляется только** при наведении на грядку или землю
- При наведении на другие блоки (камень, дерево, траву и т.д.) - **визуализация не показывается**

---

## 🔧 Логика Валидации Блоков

### ToolHoe (Мотыга)
```java
@Override
public boolean canApplyToBlock(Level level, BlockPos pos, ItemStack stack) {
    BlockState state = level.getBlockState(pos);
    BlockState aboveState = level.getBlockState(pos.above());
    if (!aboveState.isAir()) return false;
    return state.is(ModBlocks.DIRT.get()) || state.is(ModBlocks.FARMLAND.get());
}
```
- ✅ Возвращает `true` для блоков земли (DIRT) или грядок (FARMLAND)
- ❌ Возвращает `false` если сверху есть блок (не воздух)

### ToolWateringCan (Лейка)
```java
@Override
public boolean canApplyToBlock(Level level, BlockPos pos, ItemStack stack) {
    BlockState state = level.getBlockState(pos);
    BlockState aboveState = level.getBlockState(pos.above());
    if (!aboveState.isAir()) return false;
    return state.is(ModBlocks.FARMLAND.get()) && !isEmpty(stack);
}
```
- ✅ Возвращает `true` для грядок (FARMLAND)
- ❌ Возвращает `false` если лейка пустая или сверху есть блок

## 🎨 Визуальные Особенности

### Рендеринг

1. **Позиция:** Оверлей рисуется чуть выше блока (`y + 1.001`)
2. **Прозрачность:** Alpha = 200 (полупрозрачный)
3. **Смешивание:** Использует `RenderSystem.defaultBlendFunc()`
4. **Глубина:** Отключает depth test для видимости через блоки
5. **Батчинг:** Сначала рендерятся все зелёные, затем все красные блоки

### Текстуры

| Состояние | Текстура | Описание |
|-----------|----------|----------|
| Можно применить | selection_green.png | Зелёная текстура оверлея |
| Нельзя применить | selection_red.png | Красная текстура оверлея |

### Shader
```java
RenderSystem.setShader(GameRenderer::getPositionTexColorShader);
RenderSystem.setShaderTexture(0, GREEN_OVERLAY); // или RED_OVERLAY
```
- Использует стандартный шейдер с позицией, текстурой и цветом
- Белый цвет (255, 255, 255) для отображения текстуры без искажений
- Две отдельные текстуры для двух состояний

## 🚀 Производительность

### Оптимизации:

1. **Проверка только при наведении:** Рендерится только когда `HitResult.Type == BLOCK`
2. **Проверка инструмента:** Быстрая проверка `instanceof IPatternTool`
3. **Батчинг:** Все квады рендерятся одним вызовом `Tesselator.end()`
4. **Клиентская сторона:** Вся логика выполняется только на клиенте
5. **Маска визуализации:** Оверлей появляется только на грядках/земле

| FIVE | 5 | 5 | Низкая |
