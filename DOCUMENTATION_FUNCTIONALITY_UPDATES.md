# Обновление Мотыги - Докапывание Грядок

## 🔧 Что Изменилось

### ❌ Старое Поведение
- Мотыга работала **только по земле (dirt)**
- При клике по грядке (farmland) - ничего не происходило
- Визуализация показывала грядки **красным** (нельзя применить)

### ✅ Новое Поведение
- Мотыга работает **и по земле (dirt) и по грядкам (farmland)**
- При клике по грядке - "докапывает" её (проигрывает звук + эффект)
- Визуализация показывает грядки **зелёным** (можно применить)

---

## 🎯 Логика Работы

### Проверка Перед Обработкой
```
⚠️ Обязательное условие: Блок СВЕРХУ должен быть ВОЗДУХОМ
```
- Если сверху что-то есть → блок **пропускается** (показывается красным 🔴)
- Если сверху воздух → блок обрабатывается (показывается зелёным 🟢)

### 1. Клик по Земле (Dirt)
```
Dirt + Air Above → Farmland
```
- Земля превращается в грядку
- Проигрывается звук вскапывания
- Спавнятся частицы пыли
- Если идёт дождь → грядка сразу мокрая

### 2. Клик по Грядке (Farmland)
```
Farmland + Air Above → Farmland
```
- Грядка остаётся грядкой
- Проигрывается звук вскапывания
- Спавнятся частицы пыли
- Если идёт дождь → грядка поливается

---

## 💡 Польза

### Практическое Применение:
1. **Полив во время дождя** - можно "докопать" сухие грядки во время дождя, чтобы их полить
2. **Визуальная согласованность** - при использовании больших паттернов (3x3, 5x5) не нужно избегать грядок
3. **Удобство** - не нужно переключать инструмент если случайно кликнул по грядке

### Пример:
```
Ситуация: У вас есть поле с грядками, идёт дождь, но некоторые грядки сухие

Старое поведение:
- Нужно брать лейку
- Поливать вручную каждую сухую грядку

Новое поведение:
- Берёте мотыгу с паттерном 3x3 или 5x5
- Кликаете по полю
- Все грядки в паттерне автоматически поливаются дождём!
```

## 🔧 Технические Детали

### Изменённый Код:

#### canApplyToBlock() - Валидация для Визуализации
```java
@Override
public boolean canApplyToBlock(Level level, BlockPos pos, ItemStack stack) {
    BlockState state = level.getBlockState(pos);
    // Проверяем что блок сверху - воздух
    BlockState aboveState = level.getBlockState(pos.above());
    if (!aboveState.isAir()) {
        return false; // Красный оверлей если сверху что-то есть
    }
    // Можно применить к земле (dirt) или к грядке (farmland)
    return state.is(ModBlocks.DIRT.get()) || state.is(ModBlocks.FARMLAND.get());
}
```

#### useOn() - Проверка Целевого Блока
```java
BlockState targetState = level.getBlockState(clicked);
// Проверяем что целевой блок - это земля или грядка
if (!targetState.is(ModBlocks.DIRT.get()) && !targetState.is(ModBlocks.FARMLAND.get())) {
    return InteractionResult.PASS;
}
```

#### useOn() - Обработка Блоков
```java
for (BlockPos pos : pattern.getAffectedPositions(...)) {
    BlockState state = level.getBlockState(pos);
    
    // Проверяем что сверху воздух
    BlockState aboveState = level.getBlockState(pos.above());
    if (!aboveState.isAir()) {
        continue; // Пропускаем этот блок если сверху что-то есть
    }
    
    // Обрабатываем землю (dirt) - превращаем в грядку
    if (state.is(ModBlocks.DIRT.get())) {
        level.setBlock(pos, ModBlocks.FARMLAND.get().defaultBlockState(), 3);
        level.playSound(null, pos, SoundEvents.HOE_TILL, SoundSource.BLOCKS, 1.0F, 1.0F);
        
        if (level instanceof ServerLevel serverLevel) {
            spawnTillingParticles(serverLevel, pos);
        }
        
        if (isRaining && !level.isClientSide) {
            var blockEntity = level.getBlockEntity(pos);
            if (blockEntity instanceof FarmlandBlockEntity farmland) {
                farmland.hydrate();
            }
        }
    }
    // Обрабатываем грядку (farmland) - "докапываем"
    else if (state.is(ModBlocks.FARMLAND.get())) {
        // Просто проигрываем звук, блок остаётся грядкой
        level.playSound(null, pos, SoundEvents.HOE_TILL, SoundSource.BLOCKS, 1.0F, 1.0F);
        
        if (level instanceof ServerLevel serverLevel) {
            spawnTillingParticles(serverLevel, pos);
        }
        
        // Если идёт дождь, поливаем грядку
        if (isRaining && !level.isClientSide) {
            var blockEntity = level.getBlockEntity(pos);
            if (blockEntity instanceof FarmlandBlockEntity farmland) {
                farmland.hydrate();
            }
        }
    }
}
```

# Исправление Расхода Воды в Лейке

## 🐛 Проблема

Лейка расходовала **только 1 воду** независимо от количества политых блоков.

### Пример Проблемы:
```
Есть: 100 воды
Паттерн: GRID_5X5 (25 блоков)
Полив: 25 грядок

Ожидалось: 100 - 25 = 75 воды
Было: 100 - 1 = 99 воды ❌
```

---

## ✅ Решение

Исправлена логика расхода воды - теперь тратится **по 1 воде за каждый политый блок**.

### Как Работает Теперь:

```
Цикл полива:
1. currentWater = getWater(stack)  // Получаем текущее количество
2. Для каждого блока:
   - farmland.hydrate()
   - currentWater--               // Уменьшаем на 1
3. setWater(stack, currentWater)  // Сохраняем оставшуюся воду
```

---

## 🔧 Технические Изменения

### Файл: `ToolWateringCan.java`

#### Было (неправильно):
```java
if (hydratedCount > 0) {
    setWater(stack, getWater(stack) - 1); // ❌ Всегда -1
    level.playSound(null, clicked, SoundEvents.BOTTLE_FILL, SoundSource.PLAYERS, 1.0F, 1.0F);
}
```

#### Стало (правильно):
```java
if (hydratedCount > 0) {
    setWater(stack, currentWater); // ✅ Устанавливаем оставшуюся воду после полива
    level.playSound(null, clicked, SoundEvents.BOTTLE_FILL, SoundSource.PLAYERS, 1.0F, 1.0F);
}
```

### Объяснение:
- **currentWater** уже уменьшается в цикле (`currentWater--` для каждого блока)
- После цикла просто сохраняем оставшееся значение в ItemStack

## 💡 Логика Полива

### Код Цикла:
```java
int hydratedCount = 0;
int currentWater = getWater(stack);

for (BlockPos pos : pattern.getAffectedPositions(...)) {
    if (currentWater <= 0) break; // Останавливаемся если вода закончилась
    
    BlockState state = level.getBlockState(pos);
    if (!state.is(ModBlocks.FARMLAND.get())) continue;
    
    BlockState aboveState = level.getBlockState(pos.above());
    if (!aboveState.isAir()) continue;
    
    BlockEntity be = level.getBlockEntity(pos);
    if (be instanceof FarmlandBlockEntity farmland) {
        farmland.hydrate();     // Поливаем грядку
        hydratedCount++;        // Считаем политые
        currentWater--;         // Тратим воду ✅
        
        if (level instanceof ServerLevel serverLevel) {
            spawnWaterSplashParticles(serverLevel, pos);
        }
    }
}

if (hydratedCount > 0) {
    setWater(stack, currentWater); // ✅ Сохраняем оставшуюся воду
    level.playSound(null, clicked, SoundEvents.BOTTLE_FILL, SoundSource.PLAYERS, 1.0F, 1.0F);
}
```

### Ключевые Моменты:
1. **`currentWater--`** - уменьшает воду за каждый блок
2. **`if (currentWater <= 0) break`** - останавливает полив если вода закончилась
3. **`setWater(stack, currentWater)`** - сохраняет итоговое количество воды

# Проверка Воздуха Сверху

## 🌬️ Обзор

Добавлена проверка наличия воздуха над блоком для обоих инструментов - мотыги и лейки. Это соответствует поведению ванильного Minecraft и предотвращает нелогичные действия.

---

## 🎯 Изменения

### ✅ ToolHoe (Мотыга)

**Что проверяется:**
- Блок сверху должен быть **воздухом**
- Если сверху что-то есть → блок **не вскапывается**

**Примеры:**
```
Земля + воздух сверху ✅ → можно вспахать
Земля + трава сверху ❌ → нельзя вспахать
Земля + цветок сверху ❌ → нельзя вспахать
Грядка + воздух сверху ✅ → можно докопать
Грядка + сундук сверху ❌ → нельзя докопать
```

---

### ✅ ToolWateringCan (Лейка)

**Что проверяется:**
- Блок сверху должен быть **воздухом**
- Если сверху что-то есть → грядка **не поливается**

**Примеры:**
```
Грядка + воздух сверху ✅ → можно полить
Грядка + растение сверху ❌ → нельзя полить
Грядка + забор сверху ❌ → нельзя полить
Грядка + факел сверху ❌ → нельзя полить
```

---

## 🎨 Визуализация

### Цветовая Индикация:

**Мотыга:**
```
🟢 Зелёный = Земля/Грядка + воздух сверху (можно обработать)
🔴 Красный = Не земля/грядка ИЛИ есть блок сверху (нельзя)
```

**Лейка:**
```
🟢 Зелёный = Грядка + воздух сверху (можно полить)
🔴 Красный = Не грядка ИЛИ есть блок сверху (нельзя)
```

## 🔧 Технические Детали

### Код Проверки

**Одинаков для обоих инструментов:**

```java
// В canApplyToBlock() - для визуализации
BlockState aboveState = level.getBlockState(pos.above());
if (!aboveState.isAir()) {
    return false; // Показываем красным
}

// В useOn() - при обработке
BlockState aboveState = level.getBlockState(pos.above());
if (!aboveState.isAir()) {
    continue; // Пропускаем этот блок
}
```

### Метод `isAir()`

Проверяет является ли блок воздухом:
- `minecraft:air` → `true`
- `minecraft:cave_air` → `true`
- `minecraft:void_air` → `true`
- Все остальные блоки → `false`

---

## 💡 Почему Это Важно

### 🛡️ Защита Растений
- Нельзя случайно вспахать землю под растениями
- Нельзя полить грядку с растением (и утопить его)
- Декоративные элементы в безопасности

### 🎮 Реалистичность
- Поведение как в ванильном Minecraft
- Логично - нельзя копать/поливать через блоки
- Интуитивно понятно игрокам

### 👁️ Визуальная Обратная Связь
- Игрок заранее видит что будет обработано (🟢)
- Игрок видит препятствия (🔴)
- Не тратится время на бесполезные клики

---

## 🎯 Что Считается Препятствием

### ❌ Блокирует Обработку:
- Растения (пшеница, морковь, томаты и т.д.)
- Трава и цветы
- Факелы и редстоун-факелы
- Заборы и стены
- Сундуки и бочки
- Любые твёрдые блоки
- Жидкости (вода, лава)

### ✅ НЕ Блокирует:
- Воздух (minecraft:air)
- Воздух в пещерах (minecraft:cave_air)
- Воздух в пустоте (minecraft:void_air)
