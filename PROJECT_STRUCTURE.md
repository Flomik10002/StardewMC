# Структура проекта StardewMC (`src/main/java/dev/flomik/stardew/`)

## Обзор директорий

```
src/main/java/dev/flomik/stardew/
├── StardewMod.java
├── client/                     # Клиентская часть (рендер, экраны, модели)
│   ├── ClientModelRegistry.java
│   ├── ClientSeasonManager.java
│   ├── ClientSetup.java
│   ├── PatternOverlayRenderer.java
│   ├── model/                  # Кастомные модели блоков (сезонные вариации)
│   ├── renderer/               # BER (BlockEntityRenderers)
│   └── screen/                 # GUI экраны
├── core/
│   ├── command/                # Команды (отладка, управление временем/погодой)
│   ├── config/                 # Конфигурация мода
│   ├── crop/                   # Система сельскохозяйственных культур
│   │   ├── block/              # Блоки культур
│   │   ├── blockentity/        # BE культур
│   │   ├── def/                # Определения культур (CropDef)
│   │   ├── logic/              # Логика роста и сбора урожая
│   │   └── runtime/            # Трекеры активных блоков для оптимизации
│   ├── item/                   # Предметы (семена, инструменты)
│   │   └── tool/               # Логика инструментов (Pattern Tool)
│   ├── menu/                   # Меню контейнеров
│   ├── network/                # Сетевые пакеты
│   ├── player/                 # Обработчики событий игрока
│   ├── registry/               # Регистрация игровых объектов
│   │   ├── block/              # Регистрация блоков
│   │   │   ├── base/           # Базовые классы блоков
│   │   │   ├── craftables/     # Крафтовые станции (бочки, сундуки и т.д.)
│   │   │   ├── shape/          # Формы блоков (соединение текстур)
│   │   │   └── surface/        # Поверхности (земля, грядки, трава)
│   │   ├── blockentity/        # Регистрация BE
│   │   ├── item/               # Регистрация предметов и вкладок
│   │   ├── menu/               # Регистрация типов меню
│   │   └── sound/              # Регистрация звуков
│   ├── time/                   # Система времени, календаря и погоды
│   └── util/                   # Утилиты (калькулятор форм)
```

## Подробное описание кода

### Корневой пакет (`dev.flomik.stardew`)

#### `StardewMod.java`
Главный класс мода.
- **Инициализация**: Регистрирует все DeferredRegister (предметы, блоки, BE, звуки, меню, вкладки).
- **Сетевая часть**: Инициализирует `PacketHandler`.
- **Логика культур**: Вызывает `CropRegistry.bootstrapVanillaLike` для регистрации стандартных культур (томат, цветная капуста, рис).
- **Система времени**: Регистрирует задачу в `ScheduleManager` на 6:00 утра. Каждое утро для всех миров типа "overworld":
    - Обновляется погода через `WeatherSystem`.
    - Запускается `MorningPass` (полив/засыхание грядок).
    - Запускается `GrowthSystem` (рост культур).

### Пакет `client`

#### `ClientModelRegistry.java`
Управляет подменой моделей блоков в зависимости от сезона.
- **Кэширование**: Хранит мапы `ResourceLocation` для моделей травы, сухой и мокрой грядки, а также вариантов земли для каждого сезона.
- **События**:
    - `registerExtraModels`: Регистрирует все сезонные варианты моделей, чтобы они были загружены игрой.
    - `onModelBake`: Перехватывает запекание моделей. Заменяет стандартные модели блоков `GRASS`, `GRASS_FULL`, `DIRT` и `FARMLAND` на кастомные обертки (`SeasonShapeModel`, `FarmlandSeasonModel`), которые динамически выбирают модель при рендере в зависимости от текущего сезона в клиенте.

#### `ClientSeasonManager.java`
Хранит текущий сезон на стороне клиента.
- Синхронизируется с сервером через пакеты.
- Вызывает перерисовку мира (`levelRenderer.allChanged()`) при смене сезона.

#### `ClientSetup.java`
Настройка клиента FML.
- Регистрирует `BlockEntityRenderer` для крафтовых станций (ульи, прессы, бочки, сундуки).
- Назначает слой рендера (translucent) для грядок.
- Регистрирует экраны меню (`ChestScreen`).

#### `PatternOverlayRenderer.java`
Рендерит оверлей (сетку) на блоках, когда игрок держит инструмент с областью действия (мотыга, лейка).
- Использует `RenderLevelStageEvent`.
- Рисует зеленую сетку, если действие возможно, и красную, если нет.
- Поддерживает разные паттерны (1x1, 3x1, 3x3 и т.д.).

### Пакет `client.model`

#### `FarmlandSeasonModel.java`
Кастомная `BakedModel` для грядки.
- Обертывает стандартную модель.
- В `getQuads` проверяет сезон и состояние блока (сухой/мокрый).
- Накладывает оверлей "мокрости" (wet overlay) поверх сухой модели, если блок увлажнен.

#### `SeasonShapeModel.java`
Универсальная `BakedModel` для сезонных блоков (трава, земля).
- Выбирает нужную модель из map-ы на основе текущего сезона и формы блока (shape), полученной из BlockState.

### Пакет `client.renderer`

#### `ChestRenderer.java`
Рендерер для кастомного сундука.
- Рисует модель сундука (дно, крышка, замок).
- Анимирует открытие крышки.
- Поддерживает 21 вариант текстуры (цветовые вариации), выбирая текстуру на основе `BlockState` (свойство `VARIANT`).

### Пакет `client.screen`

#### `ChestScreen.java`
GUI экрана сундука.
- Отрисовывает стандартный инвентарь.
- Добавляет панель выбора цвета сундука над основным интерфейсом.
- При нажатии на цветную кнопку отправляет пакет `PacketChangeChestVariant` на сервер.

### Пакет `core.command`

#### `CommandEvents.java`
Регистратор команд мода.

#### `DateCommand.java` (`/date`)
Выводит текущую дату (день недели, число, сезон) игроку.

#### `DateTestCommand.java` (`/datetest`)
Отладочная команда для управления временем.
- `nextday`: Принудительно запускает переход на следующий день.
- `set time`: Устанавливает время суток.
- `set day`: Устанавливает день месяца (1-28).
- `set season`: Устанавливает сезон.

#### `RainTestCommand.java` (`/raintest`)
Отладка системы погоды и полива.
- Позволяет принудительно включить/выключить дождь.
- Проверяет статус погоды.

#### `RandomizeDirtCommand.java`
Рандомизирует вариации текстур блока земли в радиусе (для тестов визуального разнообразия).

#### `SoilTestCommand.java` (`/soiltest`)
Отладка состояния грядок.
- Принудительно сушит или увлажняет все грядки в мире (используя `FarmlandTracker`).
- Показывает статистику увлажненных блоков.

### Пакет `core.crop` (Система культур)

#### `CropRegistry.java`
Центральный реестр определений культур (`CropDef`).
- Позволяет искать определение культуры по ID или по семечку.
- `bootstrapVanillaLike`: Создает дефолтные культуры.

#### `FertilizerType.java`
Enum типов удобрений.
- Определяет эффекты: скорость роста, качество, удержание воды.
- Хранит силу эффекта.

#### `core.crop.block` & `blockentity`
- **`BlockCrop.java`**: Блок культуры. Обрабатывает клики (сбор урожая). Проверяет, что стоит на грядке.
- **`CropBlockEntity.java`**: Хранит состояние конкретного растения:
    - `cropId`: ID культуры.
    - `currentPhase`: Текущая стадия роста.
    - `dayOfCurrentPhase`: Сколько дней растение находится в этой стадии.
    - `fullyGrown`, `readyToHarvest`: Флаги готовности.
    - `phaseDaysAdjusted`: Локальный кэш длительности фаз с учетом бонусов скорости (удобрения, рисовые побеги).
    - Методы `growOneDay`: Логика ежедневного роста.

#### `core.crop.logic`
- **`GrowthSystem.java`**: Проходит по всем активным культурам, вызывает `growOneDay`. Обновляет визуальное состояние блока (`AGE`).
- **`HarvestHelper.java`**: Логика сбора урожая. Рассчитывает дроп, качество (с учетом удобрений и удачи), обрабатывает повторный рост (regrow).
- **`MorningPass.java`**: Утренняя обработка грядок.
    - Обезвоживает грядки.
    - Увлажняет, если идет дождь или сработал эффект водоудерживающего удобрения.
    - Удаляет культуры, которые не растут в текущем сезоне (если они не в теплице).

#### `core.crop.runtime`
- **`CropTracker.java`**, **`FarmlandTracker.java`**: Статические трекеры, хранящие списки позиций всех загруженных блоков культур и грядок. Позволяют итерироваться по ним без сканирования всего мира.

### Пакет `core.item`

#### `ItemStardewSeed.java`
Предмет семян.
- При использовании на грядке проверяет сезон и ставит блок культуры, инициализируя BE культуры нужным ID.

#### `core.item.tool`
Система инструментов с паттернами (мотыги, лейки).
- **`IPatternTool`**: Интерфейс для инструментов, имеющих режимы (1x1, 3x3 и т.д.).
- **`ToolHoe`**: Мотыга. Вспахивает землю (превращает `DIRT` в `FARMLAND`).
- **`ToolWateringCan`**: Лейка. Увлажняет грядки. Имеет запас воды.
- **`PatternProvider`** / **`PatternType`**: Реализация логики выбора блоков для разных паттернов (линия, квадрат).

### Пакет `core.network`

- **`PacketHandler.java`**: Инициализация канала сети.
- **`S2CSeasonSync.java`**: Пакет сервер->клиент для синхронизации текущего сезона.
- **`PacketChangeChestVariant.java`**: Пакет клиент->сервер для смены цвета сундука.

### Пакет `core.player`

- **`ForgeEvents.java`**:
    - Синхронизирует сезон при входе игрока.
    - Запрещает ломать сундуки, если в них есть предметы (делает их неразрушимыми).
- **`SleepEventHandler.java`**:
    - Перехватывает клик по кровати.
    - Вместо ванильного сна вызывает `ScheduleManager.forceNextDay` для мгновенного перехода на следующий день по логике мода.

### Пакет `core.registry`

#### `block.craftables`
Блоки крафтовых станций.
- **`BlockChest`** / **`BlockEntityChest`**: Кастомный сундук. 36 слотов. Хранит вариант цвета. Запрещает разрушение, если не пуст.
- **`BlockKeg`**, **`BlockBeeHouse`**, **`BlockCheesePress`**: Станции переработки. На данный момент имеют базовую реализацию с визуализацией предмета (через `VisualItemAboveRenderer`).

#### `block.surface`
- **`BlockFarmland`**: Грядка. Хранит состояния `HYDRATED` (полита), `FERTILIZER` (удобрение), `SHAPE` (форма соединения).
- **`BlockGrassSurface`** / **`BlockGrassFull`**: Трава с поддержкой соединений текстур (connected textures) через BlockState `SHAPE`.

#### `item`
- **`ModItems.java`**: Регистрация всех предметов.
- **`ModCreativeModeTabs.java`**: Вкладки креатива.

#### `sound`
- **`ModSounds.java`**: Звуки (вспашка, полив).

### Пакет `core.time`

Система календаря Stardew Valley.

- **`StardewDateData.java`**: SavedData (хранится в уровне). Содержит:
    - Текущий год, сезон, день (1-28).
    - Погоду на сегодня и завтра.
    - Удачу на сегодня.
- **`StardewClock.java`**: Тикает игровое время. Проверяет условие потери сознания (2:00 ночи) -> принудительный сон.
- **`ScheduleManager.java`**: Управляет расписанием событий.
    - `forceNextDay`: Основной метод смены дня.
        1. Генерирует погоду на завтра.
        2. Сдвигает дату.
        3. Применяет погоду к миру.
        4. Сбрасывает время на 6:00.
        5. Запускает утренние триггеры (рост растений).
- **`WeatherSystem.java`**: Детерминированная генерация погоды на основе сида мира и дня.

### Пакет `core.util`

- **`ShapeCalculator.java`**: Вычисляет форму блока (соединения) для травы и грядок, проверяя соседей (аналог Connected Textures, но на уровне BlockState).
