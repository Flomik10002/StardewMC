# Маска Визуализации

## 🎯 Обзор

Добавлена **маска для визуализации паттернов** - оверлей теперь появляется только при наведении на грядку (farmland) или землю (dirt). При наведении на другие блоки визуализация не показывается.

---

## 🎯 Что Изменилось

### ❌ До:
```
Наведение на любой блок → визуализация паттерна появляется

Примеры:
- Камень → показывает паттерн (всё красное)
- Дерево → показывает паттерн (всё красное)
- Грядка → показывает паттерн (зелёное/красное)
- Земля → показывает паттерн (зелёное/красное)
```

### ✅ После:
```
Наведение только на грядку/землю → визуализация паттерна появляется
Наведение на другие блоки → визуализация НЕ появляется

Примеры:
- Камень → НЕТ визуализации ❌
- Дерево → НЕТ визуализации ❌
- Трава → НЕТ визуализации ❌
- Грядка → показывает паттерн (зелёное/красное) ✅
- Земля → показывает паттерн (зелёное/красное) ✅
```

---

## 💡 Зачем Это Нужно

### 🎯 Фокус на Релевантных Блоках
- Визуализация появляется только там, где **может** быть применён инструмент
- Не отвлекает на нерелевантные блоки
- Сразу понятно, что этот блок не подходит для инструмента

### 🚀 Производительность
- Меньше рендеринга = лучше FPS
- Не тратятся ресурсы на отображение "красных" блоков
- Рендерится только то, что реально нужно

### 👁️ Чистый Визуал
- Нет "красного спама" при наведении на стены, камни и т.д.
- Интерфейс чище и понятнее
- Меньше визуального шума

---

## 🔧 Технические Детали

### Код Проверки

**PatternOverlayRenderer.java:**

```java
Level level = player.level();
BlockPos targetPos = blockHit.getBlockPos();

// Проверяем что целевой блок - это грядка или земля
BlockState targetState = level.getBlockState(targetPos);
if (!targetState.is(ModBlocks.FARMLAND.get()) 
    && !targetState.is(ModBlocks.DIRT.get())) {
    return; // Не показываем визуализацию для других блоков
}

// Далее - обычная логика рендеринга
```

### Логика:
1. Получаем блок, на который наведён прицел
2. Проверяем: это грядка (FARMLAND) или земля (DIRT)?
3. Если **ДА** → рендерим визуализацию паттерна
4. Если **НЕТ** → выходим из метода (ничего не рендерим)

### Импорты:
```java
import dev.flomik.stardew.core.registry.block.ModBlocks;
import net.minecraft.world.level.block.state.BlockState;
```

### Преимущества:

- ✅ Нет отвлекающих красных квадратов

- ✅ Чище интерфейс

---

## 🚀 Производительность

### Экономия Ресурсов:

**Сценарий:** Игрок ходит по пещере с мотыгой

**До:**
- Каждый блок → рендеринг паттерна (9-25 квадов для больших паттернов)
- Камень, руда, факелы и т.д. → всё рендерится красным
- Много ненужной работы GPU

**После:**
- Только земля → рендеринг паттерна
- Все остальные блоки → пропускаются
- Минимум работы GPU

### Оценка Улучшения:
```
В пещере: ~95% блоков не земля/грядка
→ Экономия ~95% рендеринга визуализации
```

# Обновление Системы Текстур

## 🎨 Что Изменилось

### ❌ Старая Система (Одна Текстура + Окрашивание)

**Было:**
- 1 текстура: `pattern_overlay.png` (белая)
- Окрашивалась в коде:
  - RGB(50, 255, 50) для зелёного
  - RGB(255, 50, 50) для красного
- Один рендер-проход с разными цветами

```java
// Старый код
if (canApply) {
    red = 50; green = 255; blue = 50;
} else {
    red = 255; green = 50; blue = 50;
}
```

---

### ✅ Новая Система (Две Текстуры)

**Стало:**
- 2 отдельные текстуры:
  - `selection_green.png` - зелёная текстура для валидных блоков
  - `selection_red.png` - красная текстура для невалидных блоков
- Текстуры используются как есть (без окрашивания)
- Два рендер-прохода с разными текстурами

```java
// Новый код
// Рендерим зелёные блоки
RenderSystem.setShaderTexture(0, GREEN_OVERLAY);
for (BlockPos pos : affectedBlocks) {
    if (tool.canApplyToBlock(level, pos, heldItem)) {
        renderBlockOverlay(poseStack, buffer, pos);
    }
}

// Рендерим красные блоки
RenderSystem.setShaderTexture(0, RED_OVERLAY);
for (BlockPos pos : affectedBlocks) {
    if (!tool.canApplyToBlock(level, pos, heldItem)) {
        renderBlockOverlay(poseStack, buffer, pos);
    }
}
```

---

## 🎯 Преимущества Новой Системы

### ✨ Гибкость Дизайна
- Можно использовать **любые** текстуры (не только цветные заливки)
- Можно добавить узоры, градиенты, иконки
- Полный контроль над визуалом через графический редактор

### 🎨 Качество
- Текстуры отображаются как задумано художником
- Нет искажений от программного окрашивания
- Белый цвет (255, 255, 255) сохраняет оригинальные цвета

### 🔧 Простота Изменения
- Замените PNG файлы - всё готово
- Не нужно менять код для изменения цветов
- Можно добавить анимированные текстуры в будущем

---

## 🔧 Технические Детали

### Изменённые Файлы:

**PatternOverlayRenderer.java:**
```java
// Было:
private static final ResourceLocation OVERLAY_TEXTURE = 
    new ResourceLocation(StardewMod.MODID, "textures/misc/pattern_overlay.png");

// Стало:
private static final ResourceLocation GREEN_OVERLAY = 
    new ResourceLocation(StardewMod.MODID, "textures/misc/selection_green.png");
private static final ResourceLocation RED_OVERLAY = 
    new ResourceLocation(StardewMod.MODID, "textures/misc/selection_red.png");
```

### Метод Рендеринга:

**Упрощён:**
```java
// Было: параметр boolean canApply, условное окрашивание
private static void renderBlockOverlay(PoseStack poseStack, BufferBuilder buffer, 
                                      BlockPos pos, boolean canApply)

// Стало: без параметра цвета, текстура определяется снаружи
private static void renderBlockOverlay(PoseStack poseStack, BufferBuilder buffer, 
                                      BlockPos pos)
```

### Батчинг:

**Улучшен для производительности:**
- Все зелёные блоки рендерятся одним батчем
- Все красные блоки рендерятся другим батчем
- Меньше переключений текстур = лучше производительность

## 🎨 Текстуры

### Требования к Текстурам:

**Формат:**
- PNG с альфа-каналом
- Размер: 16x16 пикселей (или любой квадратный)

**Расположение:**
```
src/main/resources/assets/stardew/textures/misc/
├── selection_green.png  ← Для валидных блоков (можно обработать)
└── selection_red.png    ← Для невалидных блоков (нельзя обработать)
```

### Примеры Дизайна:

**Простая заливка:**
- Однотонный зелёный/красный цвет
- Текущая реализация

**С рамкой:**
- Цветная рамка по краям
- Полупрозрачный центр

**С иконкой:**
- Центральная иконка (✓ или ✗)
- Цветной фон

**С узором:**
- Клетчатый паттерн
- Диагональные полосы
- Точки

---

## 🎮 Настройка Прозрачности

В коде (alpha channel):
```java
int alpha = 200; // 0-255
```

**Значения:**
- `255` = полностью непрозрачно
- `200` = текущее значение (полупрозрачно)
- `128` = полупрозрачно
- `0` = полностью прозрачно (невидимо)

---

## 📊 Производительность

### Сравнение:

| Метод | Переключений Текстур | Вычислений Цвета |
|-------|---------------------|------------------|
| Старый (1 текстура) | 1 | N (для каждого блока) |
| Новый (2 текстуры) | 2 | 0 |

**Результат:**
- Примерно одинаковая производительность
- Новый метод может быть быстрее на больших паттернах (GRID_5X5)
- Лучший батчинг (все зелёные → все красные)

# Обновление Цвета Частиц

## 📅 Что Изменено

Обновлён визуальный эффект вспашки для мотыги:

### Цвет Частиц:
- ❌ **Было:** Коричневый (RGB: 139, 69, 19)
- ✅ **Стало:** Желто-оранжевый **#ebab22** (RGB: 235, 171, 34)

### Тип Дополнительных Частиц:
- ❌ **Было:** ParticleTypes.POOF (облачка дыма)
- ✅ **Стало:** ParticleTypes.ASH (частицы пепла)

---

## 🔧 Технические Изменения

### Файл: `ToolHoe.java`

#### Изменение Цвета:
```java
// Было:
Vector3f brownColor = new Vector3f(0.545f, 0.271f, 0.075f);

// Стало:
Vector3f earthColor = new Vector3f(0.922f, 0.671f, 0.133f);
```

#### Изменение Типа Частиц:
```java
// Было:
level.sendParticles(ParticleTypes.POOF, ...);

// Стало:
level.sendParticles(ParticleTypes.ASH, ...);
```
